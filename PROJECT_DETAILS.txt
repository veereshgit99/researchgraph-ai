                                 VISION & IDEA
================================================================================

CORE VISION:
Build a game-changing personalized AI research companion that serves as a 
centralized hub for academic research exploration, discovery, and collaboration.

PROBLEM STATEMENT:
Researchers face challenges in:
- Discovering relevant papers across vast academic databases
- Understanding complex research concepts and methodologies
- Connecting related research areas and finding interdisciplinary insights
- Maintaining knowledge and insights from papers they've read
- Tracking research trends and receiving timely alerts

SOLUTION:
ResearchGraph AI combines:
1. A Neo4j knowledge graph storing papers, concepts, methods, authors, datasets
2. AI-powered natural language assistant powered by OpenAI GPT
3. Interactive graph visualization for research exploration
4. Persistent memory and personalization for each user

TARGET USERS:
- Academic researchers (PhD students, postdocs, professors)
- Research scientists in industry
- Students exploring research areas
- Interdisciplinary teams needing research insights

DATABASE (NEO4J):
Graph Nodes:
  Paper
    Properties: arxiv_id, title, abstract, published_date, categories, 
                pdf_url, citation_count, on_huggingface, hf_ai_keywords,
                graph_concepts, graph_methods, authors
    
  Author
    Properties: name
    
  Concept
    Properties: name, category
    
  Method
    Properties: name, algorithm_type
    
  Dataset
    Properties: name, size
    
  Metric
    Properties: name, type

Relationships:
  Paper -[:AUTHORED_BY]-> Author
  Paper -[:INTRODUCES]-> Concept
  Paper -[:PROPOSES]-> Method
  Paper -[:EVALUATES_ON]-> Dataset
  Paper -[:USES_METRIC]-> Metric

Example Query:
  MATCH (p:Paper)-[:AUTHORED_BY]->(a:Author),
        (p)-[:INTRODUCES]->(c:Concept)
  WHERE toLower(p.title) CONTAINS 'transformer'
  RETURN p, collect(a.name) as authors, collect(c.name) as concepts

================================================================================
                            HOW IT WORKS
================================================================================

USER FLOW - ASKING A QUESTION:

1. User types question in Assistant chat
   Example: "Explain the Transformer architecture"

2. Frontend sends to backend:
   POST /api/v1/assistant/chat
   {
     "message": "Explain the Transformer architecture",
     "conversation_history": [previous messages]
   }

3. Backend processes:
   a) Extract search terms using GPT-3.5-turbo
      â†’ ["Transformer", "architecture", "mechanism"]
   
   b) Query Neo4j for each term:
      - Search papers by title/abstract
      - Get related concepts and methods
      - Get author information
      
   c) Format context as markdown:
      ## Relevant Papers:
      1. **Attention Is All You Need**
         - Authors: Vaswani et al.
         - ArXiv ID: 1706.03762
         - Concepts: Attention, Self-Attention, Multi-Head Attention
         
   d) Build OpenAI request:
      System: [AI personality and capabilities]
      User: [Context] + [User Question]
      
   e) Get response from GPT-5-nano
   
   f) Format and return response with sources

4. Frontend displays:
   - Response text
   - Source badges (papers, concepts, methods)
   - Maintains message history

================================================================================
                              PERFORMANCE
================================================================================

RESPONSE TIMES:
- Neo4j query: 100-300ms
- OpenAI API call: 2-5 seconds
- Total response: 2-6 seconds (typical)

TOKEN USAGE (per query):
- System prompt: ~150 tokens
- Context (papers + concepts): 500-1500 tokens
- Conversation history: 50-100 tokens
- Response: 200-1000 tokens
- Total: 1000-3000 tokens per query

COST ANALYSIS (GPT-5-nano):
- Input: $0.05 per 1M tokens
- Output: $0.40 per 1M tokens
- Per query: ~$0.001-0.003
- 1000 queries: ~$1-3
- Very affordable! âœ“

BACKEND SETUP:
1. Navigate to backend folder
   cd backend

2. Create virtual environment
   python -m venv .venv
   .venv\Scripts\activate (Windows)

3. Install dependencies
   pip install -r requirements.txt

4. Create .env file with configuration
   OPENAI_API_KEY=sk-proj-xxx...
   NEO4J_URI=bolt://localhost:7687
   etc.

5. Start backend
   uvicorn app.main:app --reload

1. Navigate to frontend
   cd frontend/cortex-nexus-11

2. Install dependencies
